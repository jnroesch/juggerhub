4Events — create flow (stepped wizard) · event page (visitor + admin) · manage participants, contacts, co-admins, cancel
◆Mobile-first — desktop is shown for the dense manage / list screens (4e).
◆Two viewpoints this turn: Visitor (browsing, can sign up) & Admin (full control). Joined-participant & waitlisted views come later.
◆An event is teams-only OR individuals-only — never both. Chosen at creation; the limit counts whichever one.
◆Location is offline (full address incl. country) or virtual (just a link) — never both.
◆Fee is info only — the page shows the amount, the recipient (name + IBAN) and an optional deadline; participants pay by bank transfer. No in-app payment. Waitlist promotion is manual by an admin.
◆Paid events add a “pending” state: a team signs up, then holds a provisional spot as pending until it has paid and an admin approves — only then is it fully joined. Free events go straight to joined.
◆Create captures the essentials; contacts & co-admins are managed on the event page once it exists. News is a read-only feed for now (composer later).

Admin — the same page, plus Manage

There’s no separate admin layout. An admin opens the very same event page as everyone in 4c — same cover, details, participant lists, news and contacts. The only difference: where a visitor sees “Join”, an admin gets one Manage event button. It pulls up the admin hub — edit details, participants & waitlist, contacts, co-admins, post, cancel — each opening its own screen (4e, 4f).

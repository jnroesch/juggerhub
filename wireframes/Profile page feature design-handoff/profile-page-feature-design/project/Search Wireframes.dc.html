5Search / browse — one shared shell for teams · events · players. Chosen pattern: filters on demand + list rows (5a), now applied to Players (5d).
◆Where it lives: a Browse section with one page each — Teams · Events · Players — each its own search + filter page. (A single global search across all three is parked for later.)
◆All searches behave the same: live results as you type; before you type you’re browsing everything, with filters + sort already applied. What differs per entity is only the filter set & default sort — shell, interactions and states are identical.
◆Chosen pattern (5a): filters on demand — a Filters button opens a bottom sheet on mobile, a slide-over drawer on desktop — with compact list rows. 5d applies that exact shell to Players.
◆Teams (5a) & Players (5d) shown mobile + desktop. Near me / distance appears everywhere as a greyed coming soon — the hook for location later.
◆Names, cities and counts are placeholder. Players is opt-in — only players who choose to appear are searchable. Next up: Events, plus empty / no-results states.
